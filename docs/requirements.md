# AnyVersion 需求文档

## 核心概念："项目" (Project)

一个项目 = 一个可托管的开发环境单元（Node.js, MySQL, FFmpeg 等）

### 项目属性

| 属性 | 说明 | 示例 |
|---|---|---|
| id | 唯一标识 | "nodejs" |
| display_name | 显示名称 | "Node.js" |
| category | 类型 | language / tool / service |
| official_website | 官网 | "https://nodejs.org" |
| env_vars | 关联的环境变量 | NODE_PATH, NPM_CONFIG_PREFIX |
| find_rules | 本机路径解析规则 | Scoop/Chocolatey/Program Files... |
| has_cache | 是否有缓存 | true |
| cache_detect_cmd | 缓存检测命令 | "npm config get cache" |
| has_mirror | 是否支持镜像 | true |
| has_pkg | 是否有全局包管理 | true |
| pkg_manager | 包管理器名 | "npm" |
| is_service | 是否为服务 | false |
| default_port | 默认端口 | 3306 (MySQL) |

## 用户交互流程

### 流程 1：浏览项目列表
```
侧边栏 → 开发环境管理
  ├── 顶部：搜索框 + 类型筛选（全部/语言/工具/服务）
  ├── 项目卡片列表：
  │   ├── 项目名称 + 类型标签
  │   ├── 本机状态：未安装 / 已安装 vX.X / 已托管
  │   ├── 来源：Scoop / 手动 / AnyVersion
  │   └── 快捷操作：查看详情 / 托管
  └── 点击卡片 → 进入项目详情
```

### 流程 2：项目详情页
```
项目详情（Node.js 示例）
  ├── 头部：名称 + 类型 + 官网链接 + 托管状态
  ├── 子标签页（根据项目能力动态显示）：
  │   ├── 版本管理：已安装版本列表 + 远程安装 + 本地注册
  │   ├── 环境变量：相关变量及值 + 来源(HKCU/HKLM)
  │   ├── 缓存管理：缓存位置/大小 + 迁移操作
  │   ├── 镜像配置：当前源 + 切换按钮
  │   ├── 全局包：包列表 + 升级 + 官网链接
  │   └── 服务管理：启停 + 端口 + 数据目录
  └── 底部操作栏：
      ├── [托管此项目] 或 [取消托管]
      └── 操作前预览面板
```

### 流程 3：托管操作
```
用户点击 [托管此项目]
  → 弹出预览面板：
    "将要执行以下操作："
    1. 备份环境变量：GOROOT=C:\Go, GOCACHE=...  [源: HKCU]
    2. 创建目录联接：C:\Users\...\.any-version\links\go → D:\versions\go\1.22
    3. 设置环境变量：GOROOT → C:\Users\...\.any-version\links\go
    4. 将 links\go 添加到 PATH
  → 用户确认 → 执行 → 显示结果
```

## 边界情况

1. **Scoop 管理的项目**：检测到 Scoop 安装 → 提示"由 Scoop 管理" → 托管时从 Scoop 版本创建 junction
2. **环境变量冲突**：同时有 HKCU 和 HKLM 的同名变量 → 优先 HKCU → 提示冲突
3. **服务正在运行**：托管前需停止服务 → 提示用户 → 确认后操作
4. **磁盘空间不足**：安装前检查目标磁盘空间 → 不足时提示
5. **网络问题**：下载失败时保留已下载的部分 → 支持重试
6. **junction 创建失败**：权限不足时提示以管理员运行
7. **备份还原失败**：部分变量已被其他程序修改 → 跳过并报告

## 非功能需求

- **性能**：项目列表加载 < 500ms，状态扫描 < 2s，版本切换 < 1s
- **安全**：所有操作前展示预览，备份后再操作，可随时还原
- **可扩展性**：新增项目只需编辑 sdks_registry.json，无需改代码
- **透明性**：显示所有远程地址、文件路径、配置来源
- **初学者友好**：中文界面，专业术语附带说明，操作有确认步骤
