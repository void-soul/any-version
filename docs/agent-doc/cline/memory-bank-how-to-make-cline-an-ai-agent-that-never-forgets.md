        </figure><h2 id="understanding-context-windows-and-memory-in-ai-systems">Understanding Context Windows and Memory in AI Systems</h2><p>Before we dive deeper into the Memory Bank, it's important to understand why this solution is so crucial. AI assistants like Cline operate with what we call a "<a href="https://cline.bot/blog/understanding-the-new-context-window-progress-bar-in-cline?ref=cline.ghost.io">context window</a>" – essentially the amount of conversation and code they can "remember" at once. This window is finite, typically ranging from 32k to 200k tokens depending on the model.</p><p>When you're working on a complex project, you'll inevitably hit this limit. Traditional solutions include:</p><ol><li>Starting a new chat (losing all context)</li><li>Carefully managing what you include in each prompt</li><li>Using summarization techniques to compress context</li></ol><p>None of these are ideal. They either sacrifice important context or require constant manual management. The Memory Bank system offers a different approach entirely.</p><h2 id="the-architecture-of-memory">The Architecture of Memory</h2><p>Memory Bank works on two levels. files</li><li>Prepare for potential context resets</li></ol><h2 id="visual-prompting-guiding-ai-with-flowcharts">Visual Prompting: Guiding AI with Flowcharts</h2><figure class="kg-card kg-image-card"><img src="https://storage.ghost.io/c/d6/fe/d6feb101-a8e6-444b-bae8-3ca714794abb/content/images/2025/02/image-15.png" class="kg-image" alt="" loading="lazy" width="1268" height="1310" srcset="https://storage.ghost.io/c/d6/fe/d6feb101-a8e6-444b-bae8-3ca714794abb/content/images/size/w600/2025/02/image-15.png 600w, https://storage.ghost.io/c/d6/fe/d6feb101-a8e6-444b-bae8-3ca714794abb/content/images/size/w1000/2025/02/image-15.png 1000w, https://storage.ghost.io/c/d6/fe/d6feb101-a8e6-444b-bae8-3ca714794abb/content/images/2025/02/image-15.png 1268w" sizes="(min-width: 720px) 720px"></figure><p>Standing wisdom is to write system prompts in markdown, JSON, or <a href="https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/use-xml-tags?ref=cline.ghost.io">XML</a>. However, our community has found Mermaid diagrams (which Claude is quite adept at creating) to be useful for prompting.</p><p>Instead of writing process instructions markdown, they use write in flowcharts like this:</p><pre><code class="language-mermaid">flowchart TD
    Start[Start] --&gt; ReadFiles[Read Memory Bank]
    ReadFiles --&gt; CheckFiles{Files Complete?}
    
    CheckFiles --&gt;|No| Plan[Create Plan]
    Plan --&gt; Document[Document in Chat]
    
    CheckFiles --&gt;|Yes| Verify[Verify Context]
    Verify --&gt; Strategy[Develop Strategy]
    Strategy --&gt; Present[Present Approach]
</code></pre><p>Here's why this matters: Mermaid diagrams aren't just prettier than text – they're a formal language for describing workflows. Each node, connection, and decision point is explicitly defined. For an AI, this structured format might actually be easier to understand than paragraphs of natural language trying to describe the same process.</p><p>When we write text instructions, there's room for ambiguity. But a flowchart in Mermaid code is unambiguous – each state and transition is clearly defined. It's like giving the AI a precise map instead of vague directions.</p><p>The full Memory Bank instructions are in the <a href="https://docs.cline.bot/improving-your-prompting-skills/custom-instructions-library/cline-memory-bank?ref=cline.ghost.io">Cline documentation</a>.
