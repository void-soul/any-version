纯 JSON 数据仓库，无代码逻辑，被上层 AnyVersion 应用读取以驱动工具发现、安装、调度与 MCP 配置同步。
- 顶层 `providers.json` 按 `category: provider|relay` 分类列出各厂商的 OpenAI/Anthropic/Gemini 兼容 API 基地址，供统一路由到任意后端。
- 每个工具子目录（如 `claude-code/`、`codex-cli/`、`deveco/`、`gemini-cli/`、`mimocode/`、`opencode/`、`qwencode/`）包含：
  - `config.json`：工具元信息（id、displayName、avatar、nickname）、API 协议开关（supportsOpenai/supportsAnthropic/supportsGoogle）、内置模型、dispatch/resume/continue 命令模板、runner 类型（stream-json/codex-json/opencode-json）、promptMode（stdin/arg）、缓存目录、包管理器与 npm 包名、配置文件路径及字段映射（write 键值对把 AnyVersion 内部字段写入第三方格式）、会话存储类型与目录、skillsDir。
  - `paths.json`：平台相关可执行文件探测路径（win32 下 %APPDATA%/npm、scoop、bun、pnpm 等 shim），配合 detectCmd/installCmd/startCommand 完成自动发现与安装。
- 根级共享配置：
  - `mcp-config.json`：为 claude/gemini/opencode 系工具指定其 MCP 服务器配置文件路径与 format（claude/gemini/opencode），由前端「刷新」或重启后生效。
  - `skills-scan.json`：定义 baseSkillsDir（~/.agents/skills，外部 skills.sh 仓库）与各工具的 toolSkillsDirs 扫描列表，AnyVersion 自身 skill store 仍走全局 AiConfig.skills_dir。
  - `terminals.json`：Windows 终端类型枚举（cmd/powershell/wt/pwsh/wezterm/alacritty/tabby）及启动参数，附带本地代理 proxySettings（默认端口 15721）。
依赖方向单向：上层运行时只读这些 JSON，不反向修改；新增工具只需在对应子目录加 config.json + paths.json，并在 mcp-config.json / skills-scan.json 中注册即可。
