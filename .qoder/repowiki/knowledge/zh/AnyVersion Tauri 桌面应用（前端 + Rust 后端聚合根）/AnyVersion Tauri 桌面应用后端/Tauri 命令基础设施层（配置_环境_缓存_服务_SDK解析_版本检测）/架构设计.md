本模块是 `src-tauri/src/commands` 下的子目录，每个 `.rs` 文件对应一类独立的后端能力，并通过 `#[tauri::command]` 宏将函数注册为前端可调用的 RPC。

- 依赖方向：本模块内部互相调用（如 `config` ↔ `env` ↔ `cache`），并向上层 `crate::project::{registry, scanner, types}` 读取项目定义；被上层 `main.rs` 通过 `tauri::Builder::invoke_handler` 统一挂载。
- 分层职责：
  - `config.rs`：持久化 `~/.any-version/config.json` 与 `backups.json`，提供 `load_config/save_config`、RSS 源管理、存储迁移（移动 versions_dir/links_dir、重建 junction、重写 PATH 与注册表变量）。
  - `env.rs`：Windows 注册表读写（HKCU/HKLM）、PATH 智能增删、SDK 环境变量自动注入/清理、用户可配置变量、环境变量备份/恢复、`WM_SETTINGCHANGE` 广播刷新。
  - `cache.rs`：junction 创建（`mklink /J`）、带进度事件的目录复制/清理、缓存路径迁移（快路径删除+链接 vs 慢路径先拷贝再链接）。
  - `service.rs`：服务发现（tasklist/wmic/netstat/sc）、端口冲突检测、数据目录/日志目录多源解析（custom → config → env → detected → default）、启动/停止/强制终止命令渲染与执行。
  - `sdk_resolver.rs`：按优先级规则扫描 PATH/环境变量/固定路径，定位 Scoop/Chocolatey/MSYS2/Cygwin/conda/nvm/pyenv/Volta/rustup/go/winget 等 SDK 根目录，过滤 AnyVersion 管理的 links_dir。
  - `tool_version.rs`：执行 `version_cmd` 提取当前版本，从 npm/GitHub Releases/PyPI/自定义 URL 拉取最新版本，比较语义化版本号并提供升级入口。
- 跨进程通信：所有耗时操作通过 `tauri::AppHandle::emit` 推送 `migrate-progress` / `migrate-storage-progress` / `clean-cache-progress` 事件给前端 UI 展示进度。
