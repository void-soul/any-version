- 所有对前端暴露的后端能力均以 `#[tauri::command]` 标注函数，并集中写入 `lib.rs` 的 `generate_handler![]` 宏中，禁止子模块自行注册 handler
- 命令按领域拆分为 `commands/<domain>/mod.rs` 子模块（config/env/cache/service/project/ai 等），每个子模块内再按功能拆分单文件，顶层 `mod.rs` 仅做 `pub mod` 重导出
- 需要跨命令共享的异步状态（如 HTTP 服务器）通过 `tauri::App::manage(State)` 注入，而非全局变量或静态单例
- Windows 特定逻辑使用 `#[cfg(windows)]` 条件编译隔离，避免在非 Windows 平台引入 winapi 依赖
