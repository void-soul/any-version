模块由三个文件组成清晰的三层结构：
- `main.rs` 是 Windows 下隐藏控制台窗口的单行入口，仅调用 `tauri_app_lib::run()`，将实际逻辑下沉到库 crate。
- `lib.rs` 暴露 `run()` 作为唯一启动点，在 `tauri::Builder` 链式装配中完成三件事：① 启动期执行 `cleanup_legacy_env_vars()` 和 `sync_process_path()` 以修复 `windows_subsystem="windows"` 导致的 PATH 丢失并优先 AnyVersion 托管路径；② 注册 `single_instance`、`opener`、`dialog`、`updater`、`process`、`autostart`（带 `--minimized`）等插件；③ 通过 `invoke_handler!` 集中注册 `commands::*` 下的所有 Rust→JS 命令，并在 `on_window_event` / `RunEvent::ExitRequested` 钩子中处理窗口关闭与退出时代理清理。
- `tray.rs` 独立实现系统托盘：`build_tray` 创建 TrayIcon 并监听点击/菜单事件，动态扫描 `versions_dir` 下已安装版本与服务状态生成子菜单，通过 `refresh_tray_menu` 命令供前端触发重建；主窗口通过 `WebviewWindowBuilder` 懒创建并以 `decorations(false)` 无边框显示。
依赖方向单向：`main → lib → (commands, tray)`，tray 通过 `crate::commands::*` 内部调用项目/服务命令的内层函数（如 `project_use_version_inner`），避免循环依赖。
