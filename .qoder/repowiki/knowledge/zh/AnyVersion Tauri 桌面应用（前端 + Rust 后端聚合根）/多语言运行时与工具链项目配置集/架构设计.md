本模块为纯数据目录，不含任何代码；每个子目录对应一个“项目类型”（如 go、nodejs、mysql、cmake、vcpkg 等），通过一组固定命名的 JSON 文件构成统一的数据契约：
- `config.json`：项目元信息与生命周期模板，包含 id/display_name/category、bin_dirs、下载 URL 模板、version_cmd/version_parse_regex、is_service/start_cmd/service_process_exes/service_names、data_dir/log_dir/config_file、post_install 生成配置与初始化命令、conflict_managers 冲突检测、user_configurable_vars 用户可配环境变量等。
- `find_rules.json`：按 priority 排序的探测规则数组，支持 env_bin/path_contains 等 pattern 类型，用于自动定位已安装的根目录。
- `env_vars.json`：该运行时需要清理/检查的环境变量清单（含 check_type/tier）。
- `package_managers.json`：可选的包管理器扩展定义（npm/yarn/pnpm 等），复用同一套 cache/proxy/mirror/pkg_list_format 字段族。
- `remote_versions_config.json`：远程版本源 API 规范（type/url/response_type/filter_field 等），供 Node/Deno/Go/MongoDB/MySQL 等拉取可用版本列表。
依赖方向单向向外：上层运行时加载器读取这些 JSON 驱动安装/卸载/启停逻辑，本模块不反向依赖任何实现代码。
