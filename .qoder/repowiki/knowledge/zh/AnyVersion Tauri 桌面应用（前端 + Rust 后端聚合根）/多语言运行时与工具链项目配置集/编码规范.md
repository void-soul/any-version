- 每个项目子目录严格使用 config.json / find_rules.json / env_vars.json / package_managers.json / remote_versions_config.json 这组固定文件名，作为统一的声明式契约。
- config.json 中所有路径模板统一使用 `{install_root}`、`{basedir}`、`{home}`、`{port}`、`{datadir}` 等占位符，由运行时在解析时注入。
- 版本解析采用 `version_exe` + `version_cmd` + `version_parse_regex` 三元组合，正则捕获第一个分组即为版本号。
- 可选项（mirror_options、conflict_managers、data_dirs、user_configurable_vars）均以 id 标识并附带 display_name，保持 UI 层可直接渲染。
- find_rules.json 中的规则按 priority 升序匹配，priority 越小优先级越高，source_label 用于向用户展示来源名称。
