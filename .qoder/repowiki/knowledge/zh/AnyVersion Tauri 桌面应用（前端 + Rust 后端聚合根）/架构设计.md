- 顶层 `package.json` 通过 `tauri dev/build` 脚本同时启动 Vite 开发服务器（固定端口 9999，HMR 走 ws://1421）和 Tauri CLI，由 `vite.config.ts` 忽略 `src-tauri` 目录避免重复编译。
- 前端 `react_frontend` 通过 `@tauri-apps/api` 调用 Rust 侧暴露的命令；Rust 命令在 `tauri_backend/src/commands.rs` 中注册，形成 Webview ↔ Rust 的单向 IPC 契约。
- `ai_tools_config` 与 `project_configs` 作为纯 JSON 数据源被前端与 Rust 共同消费：前者提供 AI CLI 工具元数据，后者声明 Go/Node/MySQL/Deno/Flutter 等 20+ 运行时的安装与镜像规则。
- `docs_and_scripts` 的 `scripts/agent-docs-sync.mjs` 由 Node 直接执行，将各 Agent 站点文档拉取到 `docs/agent-doc`，供前端 Markdown 渲染页展示。
- 版本发布由根级 `node scripts/bump-version.js` 驱动，同时更新 `package.json` 与 Tauri Cargo 包版本，保证前后端版本号一致。
