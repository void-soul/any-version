模块由两部分组成：
- `scripts/`：独立 Python 脚本（不依赖主程序），以 `agent_doc_manifest.json` 为唯一数据源，按 `docsSource.type` 分发到五种抓取策略（`llms_txt`、`documentation_md`、`readme_only`、`documentation_scrape`、`documentation_index`），逐页 GET 后落盘到 `docs/agent-doc/<localDir>/`；运行期状态写入仓库根 `agent-docs-state.json`。
- `docs/`：静态产物目录，包含三类内容——`agent-doc/`（脚本自动生成的各 Agent 文档镜像）、`tool-config/`（claude-code/codex-cli/deveco-code/gemini-cli/mimo-code/open-code/qwen-code 等工具的原始配置/说明文档，按厂商子目录组织）、`plans/` 与 `superpowers/plans/`（设计文档）以及顶层若干分析笔记。
入口为 `update_agent_docs.py`，通过 argparse 暴露 `--status` / `--check` / `--serve` / `--manifest` 子命令，其中 `--serve` 基于标准库 `http.server.SimpleHTTPRequestHandler` 提供本地 Web 索引页和 `/api/status` JSON 接口。
